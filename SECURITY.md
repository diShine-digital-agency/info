# Security Policy

## Supported Versions

We actively maintain the latest version of each tool in this organization. Security fixes are applied to the current `main` branch.

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

If you discover a security issue in any of our repositories, contact us directly through [dishine.it](https://dishine.it/). Include as much detail as possible:

- The repository and file(s) affected
- A description of the vulnerability and its potential impact
- Steps to reproduce (if applicable)

We will acknowledge your report within 5 business days and aim to release a fix within 30 days depending on severity and complexity.

We appreciate responsible disclosure and will credit researchers who report valid vulnerabilities — unless they prefer to remain anonymous.

## Scope

This policy covers all repositories under the [diShine-digital-agency](https://github.com/diShine-digital-agency) GitHub organization.
