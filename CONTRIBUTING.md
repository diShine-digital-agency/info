# Contributing to diShine Open Source Projects

First off, thanks for taking the time. Every issue report, suggestion, and pull request helps make these tools more useful for everyone — including the clients and teams we work with every day.

## How to Contribute

### Reporting a Bug

Before opening an issue, check whether it has already been reported. If not, open one and include:

- A clear title and description
- Steps to reproduce the problem
- What you expected to happen vs. what actually happened
- Your OS, Node.js version (or Python version), and any relevant environment details

### Suggesting an Improvement

We're always open to ideas, especially when they come from real use cases. Open an issue with the `enhancement` label and describe:

- The problem you're trying to solve
- Your proposed solution or approach
- Any alternatives you considered

### Submitting a Pull Request

1. Fork the repository and create your branch from `main`
2. Keep changes focused — one fix or feature per PR
3. Update documentation if your change affects how something works
4. Make sure existing functionality isn't broken
5. Write a clear PR description explaining what changed and why

We review all PRs. Smaller, well-scoped contributions get reviewed faster.

## Code Style

Match the style of the existing code in each repo. We don't enforce strict linting rules across all projects, but consistency within a file matters.

## Questions

For general questions about our tools or agency work, reach out via mail (you can find our email [eight here](https://dishine.it/contacts/) rather than opening a GitHub issue.
